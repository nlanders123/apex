### Feature: Workout Duration Timer + Finish Button
- `finishSession()` API function — calculates duration from session start time
- Live elapsed timer in SessionLogger header (updates every second)
- "Finish Workout" button at bottom of active sessions
- No migration needed — `duration_minutes` column already existed

### Feature: Session Completion Summary
- Inline completion view after finishing workout (not a separate route)
- Shows: duration, exercise count, total sets, total volume
- Per-exercise breakdown with sets and weights
- "Done" button navigates to workout history

### Feature: Recent Meals Quick-Log
- `getRecentMeals()` API — queries last 14 days of logged meals, deduplicates by name
- "Recent" section in MealLoggerModal pick view (above saved meals)
- One-tap re-log of recent meals
- Pick view now opens if there are recent OR saved meals (previously only saved)

- Per-exercise notes: toggle via sticky note icon next to exercise name
- Notes display in past session summary view

## Decisions made
- Recent meals use 14-day lookback (not 7) for better coverage
- Notes use debounced auto-save (800ms) rather than explicit save button
- Completion view is inline state toggle, not a separate route (avoids URL complexity)
- Timer uses `setInterval` at 1s resolution (sufficient for workout timing)
